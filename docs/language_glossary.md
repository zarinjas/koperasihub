# KoperasiHub Language Glossary

Purpose: standardize Bahasa Malaysia UI wording across Public, Admin, and Member areas.

Principles:

- Use Bahasa Malaysia as the default UI language.
- Keep wording professional, clear, and familiar for cooperative staff and older users.
- Prefer short labels over formal or uncommon translations.
- Keep code identifiers, model names, route names, and other technical naming in English.
- Allow common English terms when they are more natural in Malaysian admin usage.

## Preferred English Terms

Use these English terms when they read more naturally in UI:

- `Dashboard`
- `Admin`
- `Member`
- `CMS`
- `SEO`
- `Media`
- `Audit Log`

## Documents And Forms Usage

- Public navigation and public download pages should use `Muat Turun`.
- Public download centre should prefer `Pusat Muat Turun`.
- Admin document module should use `Dokumen & Muat Turun`.
- Member document area should prefer `Dokumen Saya` or `Dokumen Penting`, depending on context.
- Member-authenticated online submissions should use `Permohonan`.
- Public form discovery can use `Direktori Borang` or `Borang Online`.
- Public membership signup should use `Permohonan Keahlian`.
- Use `Dokumen` for downloadable reference files, notices, templates, and member-facing attachments.
- Do not label the downloads module as the main application submission flow.

## Empty States

Use short, neutral copy:

- `Tiada data untuk dipaparkan.`
- `Tiada rekod ditemui.`
- `Tiada keputusan carian ditemui.`
- `Belum ada data tersedia.`
- `Tiada dokumen dimuat naik setakat ini.`
- `Tiada pengumuman tersedia buat masa ini.`

## Validation Copy

Use direct and polite wording:

- `Ruangan ini diperlukan.`
- `Sila masukkan maklumat yang sah.`
- `Sila pilih satu pilihan.`
- `Fail yang dimuat naik tidak sah.`
- `Saiz fail melebihi had yang dibenarkan.`
- `Format fail tidak disokong.`
- `Tarikh tidak sah.`
- `Sila semak semula maklumat yang diisi.`

## Permission And Access Messages

Use firm but professional wording:

- `Anda tidak mempunyai kebenaran untuk melihat halaman ini.`
- `Anda tidak mempunyai kebenaran untuk melakukan tindakan ini.`
- `Akses ditolak.`
- `Sila hubungi pentadbir jika anda memerlukan akses.`

## Writing Rules

- Prefer `anda` over more casual alternatives.
- Avoid overly formal phrasing such as `sila lengkapkan butiran yang diperlukan sekiranya berkenaan`.
- Avoid slang and shortened chat-style wording.
- Keep buttons to one or two words where possible.
- Keep success, error, and helper text concise.
